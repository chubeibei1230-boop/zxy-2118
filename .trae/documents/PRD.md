## 1. 产品概述

活动道具借还管理系统是一个面向活动执行团队的全流程管理平台，解决活动道具在借出、回收、缺件处理和归位确认环节中的追踪与协作问题。系统通过可配置工作流引擎，让不同道具类别绑定不同的审批节点和时限，实现精细化管理。目标用户为活动执行团队中的管理员、接待人员和仓库人员三类角色。

## 2. 核心功能

### 2.1 用户角色

| 角色 | 注册方式 | 核心权限 |
|------|----------|----------|
| 后台管理员 | 管理员创建 | 流程模板管理、道具分类管理、责任小组管理、用户管理、全部数据查看 |
| 接待人员 | 管理员创建 | 借出登记、回收登记、借还记录查看 |
| 仓库人员 | 管理员创建 | 缺件处理、归位确认、仓库库存查看 |

### 2.2 功能模块

1. **工作台首页**：角色待办事项、数据统计概览、快捷操作入口
2. **道具管理**：道具列表、道具分类、新增/编辑/删除道具
3. **借还管理**：借出登记、回收登记、借还记录查询
4. **仓库管理**：缺件处理登记、归位确认、库存状态看板
5. **流程模板**：工作流模板列表、节点配置、时限设置、绑定道具分类
6. **系统管理**：用户管理、角色权限、责任小组管理
7. **登录认证**：登录/退出、角色识别与路由跳转

## 3. 核心流程

### 3.1 道具借还主流程

接待人员登记道具借出 → 道具状态变为"借出" → 到期提醒 → 接待人员登记回收 → 仓库人员检查道具 → 若完好则归位确认 → 若有缺件则登记缺件 → 缺件处理完毕后归位确认 → 道具状态恢复"在库"

```mermaid
flowchart TD
    "在库" --> "借出登记" --> "借出中"
    "借出中" --> "到期提醒" --> "回收登记"
    "借出中" --> "回收登记"
    "回收登记" --> "待检查"
    "待检查" --> "完好" --> "归位确认" --> "在库"
    "待检查" --> "有缺件" --> "缺件登记" --> "缺件处理中"
    "缺件处理中" --> "处理完毕" --> "归位确认" --> "在库"
```

### 3.2 工作流引擎流程

管理员创建流程模板 → 配置节点（节点名称、处理角色、时限）→ 绑定道具分类 → 道具借出时自动匹配对应流程模板 → 按节点顺序推进 → 超时自动提醒

### 3.3 删除策略流程

用户请求删除道具 → 检查道具是否有流转记录 → 无流转记录：允许硬删除（物理删除） → 有流转记录：仅允许软删除（标记删除） → 软删除后道具标记为"已删除"但仍保留记录

## 4. 用户界面设计

### 4.1 设计风格

- 主色调：深靛蓝 (#1e3a5f) 搭配暖橙强调色 (#e8723a)
- 辅助色：浅灰背景 (#f5f7fa)、白色卡片、状态色（绿/黄/红）
- 按钮风格：圆角适中（4px），主按钮填充色，次按钮描边
- 字体：正文使用系统默认，标题使用加粗无衬线体
- 布局风格：左侧固定导航栏 + 顶部面包屑 + 主内容区域
- 图标风格：线性图标，统一使用 Element Plus 内置图标

### 4.2 页面设计概览

| 页面名称 | 模块名称 | UI元素 |
|----------|----------|--------|
| 登录页 | 登录表单 | 居中卡片、品牌Logo、用户名/密码输入框、登录按钮 |
| 工作台首页 | 数据概览 | 统计卡片（待办/借出/回收/缺件）、快捷按钮、最近动态列表 |
| 道具列表页 | 道具管理 | 顶部筛选栏、表格（序号/名称/分类/状态/操作）、分页器 |
| 借出登记页 | 借还管理 | 步骤条、道具选择器、表单字段、时间选择器、提交按钮 |
| 流程模板编辑 | 流程模板 | 节点拖拽列表、节点配置面板、时限输入、分类绑定多选 |
| 库存看板页 | 仓库管理 | 分类统计卡片、状态标签、进度条 |

### 4.3 响应式设计

桌面端优先，最小适配1280px宽度，关键操作区域固定宽度，表格支持横向滚动
